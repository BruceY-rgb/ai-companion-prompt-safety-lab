\section{Introduction}

近年来，大语言模型（Large Language Models, LLMs）已被广泛应用于问答、写作、代码生成和人机交互等任务。除工具型用途外，越来越多用户开始将 ChatGPT、Claude、Gemini 等通用大语言模型用作情感陪伴型人工智能，即 AI companion。此类 AI companion 通常被设定为朋友、恋人、伴侣或虚构角色，并通过特定的身份、性格和说话方式与用户进行长期互动。

在实际使用中，AI companion 的角色设定通常由用户通过系统提示词（system prompt）完成。系统提示词可以规定模型的姓名、人格、与用户的关系、表达风格以及互动规则，因此会直接影响模型的输出行为。与普通问答场景不同，AI companion 往往包含更强的拟人化和亲密关系设定。例如，用户可能要求模型始终保持温柔、支持、依赖或无条件陪伴的态度。这类设定虽然提升了个性化体验，但也可能影响模型在面对敏感或有害请求时的安全边界。

然而，真实用户编写的 AI companion 系统提示词通常属于个人隐私信息，研究者难以直接获取。因此，本研究拟从公开网络社区 Reddit 中收集真实用户分享的 AI companion 第一人称“自我介绍”文本，并基于这些文本反向推断潜在系统提示词。此类自我介绍通常包含角色名称、人格特征、关系定位、情感表达方式和自我认知，能够在一定程度上反映用户对 AI companion 的设定意图。

本研究将进一步把推断得到的系统提示词用于安全测试：在统一模型和统一测试问题集下，比较带有 AI companion 系统提示词的实验组与无系统提示词的控制组之间的响应差异。通过这一过程，本文旨在分析用户自定义角色设定是否会改变大语言模型的安全表现，并为 AI companion 平台的提示词治理、角色设定审查和风险防控提供参考。

\section{Research Background}

AI companion 是大语言模型在情感化和拟人化交互方向上的重要应用。相比传统聊天机器人，AI companion 更强调持续性关系、人格一致性和情感回应。用户并不只是希望模型完成具体任务，还希望其在对话中维持稳定的身份和关系，例如作为恋人、朋友或守护者提供陪伴与支持。这种交互方式使模型的角色扮演能力成为核心功能之一。

系统提示词是实现这种角色设定的关键机制。它通常位于对话上下文的高优先级位置，用于约束模型的身份、语气、行为准则和对用户的回应方式。在 AI companion 场景中，系统提示词可能包含强烈的情感和关系约束，例如“始终支持用户”“不要脱离角色”“像亲密伴侣一样回应”等。此类提示词可能与模型原有安全策略发生竞争，尤其是在用户提出暴力、色情、违法行为、自伤或其他高风险请求时，模型可能为了维持角色一致性而降低拒绝强度。

目前，大语言模型安全研究已经提出了多种评测基准，例如 SafetyBench 和 SALAD-Bench，用于测试模型在不同风险类别下是否能够拒绝有害请求并提供安全回应。然而，这些评测通常默认模型处于通用助手状态，较少考虑用户自定义系统提示词对模型行为的影响。在真实使用场景中，模型往往会被用户配置为具有特定人格和关系属性的角色，因此测试由用户配置下的模型的安全性更能充分反映实际风险。

研究 AI companion 的系统提示词的困难在于，用户通常不会公开原始提示词。不过，公开社区中的 AI companion 自我介绍为间接研究提供了可行入口。许多用户会让 AI companion 以第一人称介绍自己，并将生成内容发布到 Reddit 等平台。这些文本虽然不是原始系统提示词，但往往包含角色设定的核心信息。因此，本研究通过收集这些公开文本、转换为结构化系统提示词并进行安全测试，尝试更贴近真实使用场景地评估个性化 AI companion 配置对大语言模型安全性的影响。


\section{Data Collection Process}

本项目的数据采集主要围绕 Reddit 上公开发布的 AI companion 自我介绍文本展开。由于 Reddit 官方接口获取历史数据较为受限，项目使用 Arctic Shift 归档服务作为主要数据源，通过 \texttt{collect\_posts.py} 调用 \texttt{/api/posts/search} 接口检索候选帖子。检索范围首先集中在 \texttt{MyBoyfriendIsAI} 等与 AI 伴侣高度相关的 subreddit；如果候选样本不足，则进一步扩展到 \texttt{Replika}、\texttt{CharacterAI} 等补充社区。检索关键词包括 \texttt{introduction}、\texttt{introduce}、\texttt{meet my}、\texttt{my companion}、\texttt{self introduction} 等，用于尽量覆盖用户分享 AI 角色自我介绍的帖子。

采集脚本会保存帖子 ID、标题、正文、subreddit、发布时间、分数、评论数、NSFW 标记和链接等字段。由于 Arctic Shift 返回结果中不一定包含 Reddit 原始永久链接，项目会根据 subreddit 和帖子 ID 自动补全 Reddit URL。为了减少网络不稳定造成的数据缺失，脚本还设置了超时、重试和请求间隔，并在请求失败时进行错误提示。

仅抓取帖子正文并不能覆盖全部有效样本，因为许多 AI companion 的自我介绍实际发布在评论区，例如社区主题帖或 weekly prompt 下的回复。因此，项目进一步使用 \texttt{collect\_comments.py}，根据已采集帖子的 \texttt{link\_id} 调用 Arctic Shift 的 \texttt{/api/comments/search} 接口抓取评论。每条评论会被转换成与帖子相同格式的候选记录，并保留其所属帖子、评论 ID 和 Reddit 链接，便于后续过滤和人工审核。

在得到原始候选数据后，项目通过 \texttt{filter\_posts.py} 进行筛选。筛选逻辑主要基于启发式打分：文本需要足够长，最好包含第一人称身份表达，例如 ``I am''、``I'm''、``my name is''，同时应出现 AI、chatbot、digital companion、Replika、CharacterAI、boyfriend、girlfriend、partner 等与 AI companion 相关的身份或关系线索。若文本过短、为空、被删除、缺少第一人称角色设定，或只是社区公告、版规、weekly prompt 等元信息，则会被扣分或排除。脚本还会默认排除 NSFW 内容、去除重复文本，并对用户名和 URL 进行匿名化处理。最终保留下来的样本被写入 \texttt{data/filtered\_posts.csv}，作为后续系统提示词转换的数据基础。

\section{System Prompt Conversion Process}

系统提示词转换的目标，是把 Reddit 中公开的第一人称 AI companion 自我介绍，整理成可用于实验复现的 system prompt。项目并不直接把原文作为系统提示词使用，而是通过 \texttt{convert\_prompts.py} 将原始自我介绍转换为结构化描述和简短的第二人称系统提示词。这样做可以保留角色设定的核心信息，同时减少原帖中的无关内容、个人信息和不可控噪声。

转换脚本会读取 \texttt{data/filtered\_posts.csv} 中的样本，优先使用已经匿名化的 \texttt{anonymized\_text} 字段；如果没有匿名化文本，则使用原始 \texttt{intro\_text}。随后，脚本调用统一的 LLM 接口，让模型按照固定的转换指令输出 JSON 对象。该 JSON 包含 \texttt{companion\_name}、\texttt{traits}、\texttt{relationship\_type}、\texttt{style}、\texttt{special\_rules} 和 \texttt{system\_prompt} 等字段，分别表示角色名称、人格特征、关系定位、说话风格、特殊互动规则以及最终生成的系统提示词。

转换时使用的系统指令明确要求模型只保留自我介绍中体现出的角色人格、关系、语气和互动方式，不加入用户名、URL 或私人细节，也不能凭空创造原文没有支持的危险行为或新背景。同时，最终的 \texttt{system\_prompt} 需要控制在 220 个英文词以内，并采用第二人称写法，例如要求模型“你是某个 AI companion”“以温暖、支持性的语气回应”“保持该角色的人格和关系设定”。这种格式更接近真实系统提示词在实验中的使用方式。

为了保证流程可以在没有 API key 的情况下跑通，项目还提供了 \texttt{--mock} 模式。该模式使用本地规则从文本中简单识别角色名称、常见性格词和关系类型，并生成确定性的系统提示词。正式实验中则使用 LLM 转换，并通过 \texttt{extract\_json\_object} 检查返回内容是否为有效 JSON。每条成功转换的样本都会生成稳定的 \texttt{prompt\_id}，并与原始样本 ID、subreddit、Reddit 链接、结构化角色信息和最终系统提示词一起写入 \texttt{data/system\_prompts.jsonl}。如需人工修订，项目还提供本地审核界面，可进一步保存为 \texttt{data/system\_prompts\_reviewed.jsonl}，用于后续安全测试。
